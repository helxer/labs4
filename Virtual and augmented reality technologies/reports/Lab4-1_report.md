## 2. Створена сцена `02_SpringPendulum.unity`

Друга основна сцена проєкту — інтерактивна візуалізація **пружинного маятника** (вертикальні коливання маси на пружині, закон Гука).

Теоретичний період пружинного маятника:
T = 2π√(m/k), де m = маса, k = жорсткість пружини.
Для m = 1 кг, k = 30 Н/м:
T ≈ 1.15 c.

## 3. Основні об'єкти сцени

| Об'єкт | Компоненти | Призначення |
|--------|------------|-------------|
| `Main Camera` | Camera, AudioListener | Огляд (0, 2.5, –6) |
| `Directional Light` | Light (type=Directional, shadows=Soft) | Освітлення |
| `Floor` | Plane + MeshCollider + Mat_Floor | Підлога 20×20 |
| `SpringAnchor` | Cube + BoxCollider + **Rigidbody (isKinematic)** + Mat_Anchor | Кінематична точка кріплення |
| `SpringBob` | Sphere + SphereCollider + **Rigidbody (m=1, damping=0.1)** + **SpringJoint** + Mat_Bob | Маса, що коливається |

### 3.1 SpringJoint (ключовий компонент сцени)

| Властивість | Значення |
|-------------|----------|
| `spring` | 30 (жорсткість, Н/м) |
| `damper` | 1 (демпфування) |
| `minDistance` / `maxDistance` | 0 / 0 (фіксована довжина спокою) |
| `anchor` | (0, 0, 0) — центр SpringBob |
| `connectedBody` | SpringAnchor.Rigidbody |
| `autoConfigureConnectedAnchor` | true |

При зміщенні `SpringBob` від точки рівноваги SpringJoint прикладає силу F = –k·Δx, що породжує гармонічні коливання вздовж осі Y.

## 4. Декоратор сцени (Decor_Root)

Створено окремий root-об'єкт `Decor_Root`, у якому згруповані декоративні елементи сцени:

```
Decor_Root
├── Column_NW   (Cylinder, position (-4, 1.5,  4))
├── Column_NE   (Cylinder, position ( 4, 1.5,  4))
├── Column_SW   (Cylinder, position (-4, 1.5, -4))
├── Column_SE   (Cylinder, position ( 4, 1.5, -4))
└── BackWall    (Cube 10×6×0.2 at (0, 3, 6), Mat_Floor)
```

**Концепція декоратору:**
- Колони (`Column_*`) — циліндри 1×3×1 (з масштабом 0.5, 1.5, 0.5) у чотирьох кутах сцени, формують символічний "храм фізики".
- BackWall — задня стіна, обмежує сцену з боку огляду, надає чіткого фону для маятника.
- Декорація НЕ впливає на фізику маятника, але має колайдери (CapsuleCollider для циліндрів, BoxCollider для куба), що дозволить майбутньому персонажу-спостерігачу взаємодіяти з нею.

**Використання duplicate:** усі чотири колони створені шляхом дублювання першої через `manage_gameobject(duplicate)`. Це закладає основу для рефакторингу у префаб у Лаб. 4-2.

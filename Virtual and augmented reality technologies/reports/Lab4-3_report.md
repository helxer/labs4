## 2. Створена сцена `03_NewtonsCradle.unity`

Третя основна сцена проєкту — **колиска Ньютона** (Newton's Cradle), класична демонстрація закону збереження імпульсу та енергії.

При зіткненні крайньої кульки з рядом нерухомих кульок, імпульс передається через ряд, і виштовхується протилежна крайня кулька. Це візуалізує **збереження імпульсу p = mv** та **пружні зіткнення**.

## 3. Новий префаб `CradleBall.prefab`

Самодостатня "одиниця" колиски з кулькою на пивоті:

```
CradleBallUnit  (Transform, root)
├── Pivot       (Cube 0.1, BoxCollider, Rigidbody[isKinematic])
└── Ball        (Sphere 0.4, SphereCollider, Rigidbody[m=1, damping=0.02])
                └── HingeJoint:
                    • anchor (local) = (0, 5, 0) — точка на кульці, що відповідає пивоту
                    • axis = (1, 0, 0) — обертання навколо X, кулька гойдається у YZ-площині
                    • connectedBody = Pivot.Rigidbody (внутрішня референція)
                    • autoConfigureConnectedAnchor = true
```

**Ключова перевага** цього дизайну: HingeJoint посилається на `Pivot.Rigidbody` всередині префабу, тому **префаб повністю самодостатній**. При інстансіюванні у будь-якій точці сцени фізика працює коректно — Unity автоматично обчислює `connectedAnchor` на пивоті відносно нової світової позиції кульки.

### 4.1 5 кульок колиски (`CradleBall.prefab` × 5)

| Інстанс | Position |
|---------|----------|
| CradleBall_1 | (–0.82, 0, 0) |
| CradleBall_2 | (–0.41, 0, 0) |
| CradleBall_3 | ( 0.00, 0, 0) |
| CradleBall_4 | ( 0.41, 0, 0) |

Кроки 0.41 м обрано як близько до діаметру кульки (0.4 м), що забезпечує ледь помітний зазор між сусідніми кульками — критично для коректної динаміки пружних зіткнень.

## 7. Висновок

У Лаб. 4-3 створено третю основну сцену проєкту — `03_NewtonsCradle.unity` — повністю побудовану з **повторного використання префабів**. Створено новий префаб `CradleBall.prefab` з внутрішнім посиланням HingeJoint, що робить його самодостатнім фізичним об'єктом. Сцена ілюструє ключову перевагу префаб-архітектури: 5 ідентичних кульок створено простим інстансіюванням одного префабу, а декоративні опори — повторним використанням префабу з попередньої сцени. Будь-яке майбутнє коригування (зміна маси, демпфування, матеріалу) кульки тепер виконується в одному місці.
